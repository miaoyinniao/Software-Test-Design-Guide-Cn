介绍

等价类划分测试是最基本的测试设计技术。他帮助测试人员选择可能覆盖最全的用例最小子集。等价类划分有另外一个用处，他引导我们进行边界值测试。

在前一章节中，以下规则是指说明我们怎么根据年龄来确认是否雇用的程序。规则如下：

0–16

Don't hire

16–18

Can hire on a part-time basis only

18–55

Can hire as a full-time employee

55–99

Don't hire

注意边界中的问题，16在两个等价类中都存在，但结果不一致。
边界值测试聚焦于边界因为在这些地方问题大量发生。有经验的测试人员深有体会，初学者有直觉认为问题会在这些地方出现。这些缺陷可以在需求中就存在也可能在代码中。：
关键点： 边界值测试聚焦于边界因为在这些地方问题大量发生










      If (applicantAge >= 0 && applicantAge <=16)








                hireStatus="NO";

当然，开发人员的错误不同于测试错误，本来应该写大于的写成了大于等于是一个例子。



最有效的发现这些缺陷的方法是检视需求或者代码。Glib和Graham的书《Software Inspection 》对这个检视流程做了详细介绍。但是不管检视多有效，我们需要测试来验证正确性。

有可能的输入边界值如： {-1, 0, 1}, {15, 16, 17}, {17, 18, 19}, {54, 55, 56}, and {98, 99, 100}. 



其他值象 {-42, 1001, FRED, %$#@} 取决于模块文档的前置条件如何描述。
